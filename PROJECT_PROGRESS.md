# Internet Shop (FastAPI)

## Цель проекта

Изучить backend-разработку на Python через создание полноценного интернет-магазина.
Главная цель — понимать архитектуру и причины принятых решений, а не просто получить работающий код.

- Используем Repository Pattern.
- Используем Service Layer.

- Изменения структуры БД только через Alembic.
- Пользователь никогда не отправляет служебные поля (id, created_at, is_deleted и т.д.).
- SQLAlchemy модели и Pydantic схемы — разные сущности.

# Учебные договорённости

## Формат работы

Проект разрабатывается в формате обучения backend-разработке.

Перед написанием нового кода:

1. Обсуждаем проблему.
2. Рассматриваем варианты архитектурного решения.
3. Пользователь предлагает свой вариант.
4. Обсуждаем плюсы и минусы решения.
5. Только после этого пишем код.

Главная цель — понимание причин решений, а не получение готового кода.

---

## Подход к написанию кода

Если задача небольшая или средняя:

- пользователь сначала самостоятельно пишет решение;
- ChatGPT выступает в роли code reviewer;
- анализируются:
  - сильные стороны решения;
  - ошибки;
  - возможные улучшения;
  - соответствие архитектуре проекта.

Код не переписывается сразу полностью без обсуждения.

---

## Обучение архитектуре

Новые паттерны и слои добавляются только после обсуждения проблемы, которую они решают.

Например:

Repository Pattern изучается через проблему смешивания:
- HTTP-логики;
- SQL-запросов;
- работы с базой.

Service Layer вводится после понимания роли Repository и необходимости отделения бизнес-логики.

---

## Логирование

Логирование добавляется осознанно.

Не логировать каждое действие подряд.

Перед добавлением логов обсуждается:
- какую проблему решает лог;
- кому он нужен;
- не создаёт ли он лишний шум.
